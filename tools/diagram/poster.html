near-zeroDistributed / Always On AG — 2016+ · needs AD DS · ports

→ Arc SQL MI / Container

Arc SQL MI exposes a SQL MI endpoint → SQL MI methods apply (backup/restore)

→ SQL database in Fabric

offlineFabric Migration Assistant
schema via DACPAC ≤ 20 MB · data via Fabric Data Factory copy + on-prem gateway (no VNet / Private Link)
Multi-TB / limited WAN (any target) — seed-then-sync: ship the initial full backup via Data Box (or AzCopy over ExpressRoute), then catch up the delta with LRS / MI Link / transactional replication / log shipping before cutover. Don't size the cutover as size ÷ bandwidth — test AzCopy + one full backup first, and plan a rollback window.
C

Cutover — downtime class, blockers & remediations

~60% of migrations stumble on undocumented linked servers / SQL Agent jobs — map dependencies first
Near-zero (minutes)
MI Link · Distributed / Always On AG · Striim
Minimal (tens of min)
transactional replication · log shipping · native restore + diff/log
Offline (planned window)
native backup/restore · LRS · BACPAC · bcp · detach/attach · ADF · Data Box
TDE-encrypted DB
Migrate the server-level certificate FIRST, before any native restore to MI — else the restore fails ~80% in, no clear error.
Windows logins
DMS skips them by default — enable the option, grant MI read to Entra ID, script + recreate logins/users.
MI Link ports
Open 5022 both directions (often blocked in banking / industrial nets).
Other methods' network
Outbound 443 to Blob, 1433 (+ 1434/UDP for named instances).
Distributed AG
Requires AD Domain Services (or workgroup AG + certs).
Txn replication → SQL DB
Source 2016–2019 only; article-type limits (no hierarchyid, sql_variant…).
SQL Agent jobs
MI: native · SQL DB: Elastic Jobs.
Linked servers / cross-DB
OK on VM / MI · not on SQL DB — refactor.
SSIS · SSRS · SSAS
Azure-SSIS IR · Power BI paginated (RDL) · AAS / Power BI (XMLA).
Pre-cutover validation:  run DEA (Database Experimentation Assistant) capture + replay for performance-sensitive workloads.

Cost levers, Microsoft program & the assessment tool to run next

Cost levers

Azure Hybrid Benefit — SQL DB (vCore) · SQL MI · SQL VM  not Fabric SQL DB
ESU — free on Azure VM (3 yrs) · on-prem only via Azure Arc (paid) — changes the stay-vs-migrate math
Sizing — never average CPU: Perfmon baseline ≥ 7 days + ~20% headroom
AHB + reservations + ESU → up to ~85% savings

Assess / orchestrate (control plane)

SSMS 22 Migration Component — DBA-first, Windows
SQL migration in Azure Arc — Copilot: MI Link · LRS · SQL VM (GA)
Azure Migrate — estate discovery / business case
modern DMS + Az.DataMigration — scale / CI-CD (beyond ~50 DBs)
SSMA — heterogeneous sources only (Oracle/Sybase/DB2…)
